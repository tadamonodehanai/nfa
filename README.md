# 長野県サッカー協会 サイトリニューアル｜情報探索UI プロトタイプ

委員会ごとに分かれた情報を、**ユーザーの“知りたい”からつなぐ**探索面（Discovery Surface）の動くプロトタイプです。「発信の構造」から「探索の構造」へ ── その体験を、実際に触って確かめられます。

## ▶ デモを見る（クライアント確認用）

### 👉 https://tadamonodehanai.github.io/nfa/

PC・スマホどちらのブラウザでもそのまま動きます（インストール不要）。

---

## 見どころ（この順で触ると伝わります）

1. **何も入力しない状態** … 委員会の区分は表に出さず、「だれの情報？（年代）」「試合・大会のこと」などのタグから探し始められる。
2. **左のタグから `小学生（4種）` を選ぶ** … 右に小学生の情報が集まる。
3. **`小学生の試合の結果` へ絞り込み** … さらに `最新節の結果／地区別の結果` へと段階的にドリルダウン。横には「ほかの年代」への派生も。
4. **検索に「試合」だけ入力** … 候補が多いと「年代を選んで」とやさしく聞き返す（＝対話）。
5. **「この結果ページを開く」** … 出典ページへ。ページ下部で「このAI案内はこのページを根拠にしている」ことを明示（＝正確性の見える化）。

## 画面

- **PC**：左右2カラム。左＝タグ探索ゾーン／右＝検索＋結果。全幅で“絞り込まれていく”動きを体感。
- **スマホ**：縦積み（検索 → タグ → 結果）。

## このデモについて（サンプルであること）

- 委員会名・チーム名・スコア・日付は**すべてサンプル**です。実データではありません。
- タグの言葉づかいや絞り込みの段階、配色・レイアウトは、ご確認いただきながら調整していきます。
- AIの案内は、本番では**協会の掲載ページを根拠にした回答（出典必須・鮮度管理）**として実装します。

## ファイル構成

| ファイル | 中身 |
| --- | --- |
| `index.html` | 画面と探索ロジック（UIの雛形） |
| `data.js` | 掲載コンテンツとタグの語彙（**ここを編集すれば中身が変わります**） |
| `README.md` | この説明 |

> 補足：`data.js` の `CONTENT` が各ページのデータ、`AGES / INTENTS / …` がタグの語彙です。本番は、コンテンツ監査で定めた4軸メタ（種別・情報の種類・目的・発信元＋優先度・出典URL）で構造化した実データに差し替えます。
